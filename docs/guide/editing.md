# Editace

Tahle část vysvětluje, jak se dokumentace upravuje v Gitu i v GitHub UI.

## Edit this page

Každá stránka může mít tlačítko pro editaci. Odkazuje přímo na příslušný Markdown soubor v repozitáři a zjednodušuje drobné opravy.

## Pracovní postup

Nejčistší postup je: upravit Markdown, zkontrolovat náhled, commitnout změnu a nechat GitHub Actions provést build.

## Verzování

Každý větší balík změn má být oddělený do samostatného commitu. Dlouhodobě se tím výrazně zlepší čitelnost historie.
